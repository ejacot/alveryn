Generated from current Flyway migrations and JPA entities. Schema version: V36.

Current architecture: WorkRecord groups one day/job, WorkRecordLine stores performed activities, and WorkType is now both formula and optional parent category. `unit_types`, `unit_entry_items`, `work_type_configurations`, `work_entries`, Excel import tables and `time_spans` are removed from the current schema.

Main relationships

  • user_accounts 1 → 1 user_profiles
  • user_accounts 1 → 1 user_preferences
  • user_accounts 1 → many addresses, work_types, work_records, absence_types, absences, hourly_rate_periods
  • addresses 1 → many work_records; optional user_profiles.address_id
  • work_types.parent_work_type_id self parent → children work_types.id
  • work_records 1 → many work_record_lines
  • work_record_lines.work_type_id many → 1 work_types.id
  • absences.absence_type_id many → 1 absence_types.id

Important current notes

  • Current work flow: create a work_record, add multiple work_record_lines, each line points to a work_type.
  • Advanced work types: a parent work_type has composite_enabled = true and child work types through parent_work_type_id.
  • Simple work types: no parent and no children; formula lives directly on work_types.
  • Absences: paid/unpaid and paid hours are configured in absence_types, then snapshotted into absences.
  • Periods: dates and times now live directly on owning tables; there is no central time_spans table.
  • Dropped by V34/V36: work_entries, time_entry_details, excel_import_batches, unit_types, unit_entry_items, work_type_configurations, and time_spans.
